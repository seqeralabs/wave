# Wave API v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a stable Wave v1 REST API under the `/w1/*` prefix, generated from a TypeSpec source of truth in a new `wave-api-v1` module, with the existing service layer reused unchanged and existing `/v1alphaN` endpoints kept as deprecated.

**Architecture:** New Gradle module `wave-api-v1` owns the TypeSpec sources (migrated from the existing `typespec/` directory which currently feeds docs only) and compiles them to versioned OpenAPI YAML. The main `wave/` app then runs the `io.micronaut.openapi` Gradle plugin to generate Java server interfaces and DTOs from that YAML; generated sources are committed back into `wave-api-v1/src/main/java/io/seqera/wave/api/v1/{spec,model}/` so downstream clients consume them without running TypeSpec. New `*V1Controller` Groovy classes implement the generated interfaces and delegate to existing services via thin mapper classes. An `AlphaDeprecationFilter` stamps `Deprecation: true` + `Sunset` headers on legacy paths.

**Tech Stack:** TypeSpec 0.64.0 (compiler/http/rest/openapi3/versioning), `io.micronaut.openapi` Gradle plugin 4.5.4, Micronaut 4.10.6, Groovy/Java 21, Spock 2 tests, `io.seqera.wave.java-library-conventions` Gradle convention plugin.

**Reference design:** `docs/superpowers/specs/2026-05-20-wave-api-v1-design.md`

**Reference codebases:**
- `~/Projects/sched/sched-api` — TypeSpec source of truth, multi-version
- `~/Projects/sched/sched-app/build.gradle` lines 165-184 (openapi config), 347-401 (clean+copy tasks)
- `~/Projects/app-bootstrap/api` and `~/Projects/app-bootstrap/app/build.gradle` — minimal v1 example

**Design-doc divergence:** During exploration we discovered the repo already has a working `typespec/` directory (TypeSpec 0.64.0, multi-file structure: `main.tsp` → `routes.tsp` → `models/*.tsp`) used for Swagger UI docs only. The design doc described a single-file structure matching sched-api. The plan keeps the existing multi-file layout because it is already authoritative and well-organised; we migrate that tree into `wave-api-v1/spec/` rather than recreating it. The design doc paragraph "TypeSpec spec organization" is corrected as part of Task 0.

```groovy
// ----------------------------------------------------------------------------
// API Generation: TypeSpec to OpenAPI Conversion
//
// Compiles spec/main.tsp into versioned OpenAPI YAML at build/openapi/.
// The latest version is copied to openapi-latest.yaml for stable downstream
// references. The main `wave/` build then runs io.micronaut.openapi against
// openapi-latest.yaml to generate Java interfaces and DTOs.
// ----------------------------------------------------------------------------

tasks.register('installTypeSpec', Exec) {
    group = 'api generation'
    description = 'Install TypeSpec compiler and dependencies from package.json'
    commandLine 'sh', '-c', 'npm install'
    inputs.file('package.json')
    inputs.file('package-lock.json').optional()
    outputs.dir('node_modules')
}

tasks.register('generateOpenApi', Exec) {
    group = 'api generation'
    description = 'Compile TypeSpec sources into versioned OpenAPI YAML and write openapi-latest.yaml'
    dependsOn 'installTypeSpec'

    commandLine 'sh', '-c', '''
        npm run typespec:compile \\
        && find build/openapi -maxdepth 1 -name 'openapi-*.yaml' -not -name 'openapi-latest.yaml' \\
           | sort -V | tail -1 \\
           | xargs -I {} cp {} build/openapi/openapi-latest.yaml
    '''.stripIndent()

    inputs.files(fileTree('spec') { include '**/*.tsp' })
    inputs.files('tspconfig.yaml', 'package.json')
    outputs.dir('build/openapi')
}

clean {
    delete 'build/openapi'
    delete 'node_modules'
}
```

import io.micronaut.http.HttpRequest
import io.micronaut.http.client.HttpClient
import io.micronaut.http.client.annotation.Client
import io.micronaut.test.extensions.spock.annotation.MicronautTest
import jakarta.inject.Inject
import spock.lang.Specification
import spock.lang.Unroll

@MicronautTest
class V1AlphaParityIntegrationSpec extends Specification {

    @Inject @Client('/') HttpClient client

    @Unroll
    def 'GET #v1Path returns the same body as GET #alphaPath'(String alphaPath, String v1Path) {
        when:
        def alphaResp = client.toBlocking().exchange(HttpRequest.GET(alphaPath), Map)
        def v1Resp    = client.toBlocking().exchange(HttpRequest.GET(v1Path),    Map)

        then:
        v1Resp.status.code == alphaResp.status.code
        v1Resp.body() == alphaResp.body()

        where:
        alphaPath              | v1Path
        '/service-info'        | '/w1/service-info'
        // Other routes need a pre-seeded build/scan/mirror id; expand as service fixtures land.
    }
}

```groovy
package io.seqera.wave.filter

import groovy.transform.CompileStatic
import io.micronaut.core.async.publisher.Publishers
import io.micronaut.http.HttpRequest
import io.micronaut.http.MutableHttpResponse
import io.micronaut.http.annotation.Filter
import io.micronaut.http.filter.HttpServerFilter
import io.micronaut.http.filter.ServerFilterChain
import org.reactivestreams.Publisher

/**
 * Stamp Deprecation + Sunset headers on the response of any pre-v1 (alpha or unversioned) API path.
 *
 * Reason: the alpha paths remain functional for in-the-wild Nextflow/Tower clients but are no longer
 * the documented surface. The headers make client maintainers aware they should move to /w1/*.
 */
@Filter([
    '/v1alpha1/**',
    '/v1alpha2/**',
    '/v1alpha3/**',
    '/container-token',
    '/container-token/**',
    '/service-info',
    '/validate-creds',
    '/scans',
    '/inspect'
])
@CompileStatic
class AlphaDeprecationFilter implements HttpServerFilter {

    private static final String SUNSET_VALUE = 'Sat, 31 May 2027 23:59:59 GMT'

    @Override
    Publisher<MutableHttpResponse<?>> doFilter(HttpRequest<?> request, ServerFilterChain chain) {
        return Publishers.map(chain.proceed(request)) { MutableHttpResponse<?> resp ->
            resp.header('Deprecation', 'true')
            resp.header('Sunset', SUNSET_VALUE)
            return resp
        }
    }
}
```

The 12-month placeholder sunset date is May 31 2027 (~12 months from the design doc date 2026-05-20). Stakeholders may adjust before GA.
