title: Docker and Singularity
description: Runtime compatibility for Docker, Podman, Singularity, and Apptainer.
---

Wave provisions images for both the Docker and Singularity container ecosystems. Docker-format images ship through a standard OCI registry interface. Singularity images ship natively in SIF format. Both formats come from the same build pipeline.

## Docker containers

Wave builds containers that are compatible with standard Docker clients. They also work with any runtime that implements the [Docker registry v2 API](https://docs.docker.com/registry/spec/api/).

Images built through Wave run on:

- Docker
- Podman
- Sarus
- Charliecloud
- Shifter

## Singularity containers

Singularity and Apptainer use a proprietary container format called the Singularity Image Format (SIF). Wave builds containers directly in SIF from a Singularityfile or from a Conda environment specification. A Docker-to-SIF conversion step is not required.

SIF images produced by Wave are stored as OCI Registry as Storage (ORAS) artifacts. You can publish these artifacts to the Wave community registry or to any OCI-compliant registry.

:::note
Freeze mode is required when provisioning Singularity images. SIF images are monolithic and do not support dynamic layer injection, so the image must be built in full and pushed to a registry.
:::

### The ORAS protocol

Wave uses the [OCI Registry as Storage (ORAS)](https://oras.land/) protocol to manage and distribute SIF containers. ORAS allows SIF images to reside in OCI-compliant registries alongside Docker images. Singularity and Docker share the same registry infrastructure.

Advantages of ORAS with Nextflow:

- **Improved resource utilization**: Singularity pulls container images inside each process task instead of the Nextflow head job. This reduces head-job pressure and improves task parallelization.
- **Private registry access**: Singularity supports authenticated pulls from private registries. See [Singularity registry](https://docs.sylabs.io/guides/main/user-guide/cli/singularity_registry.html).

Considerations and limitations:

- **Shared cache limitations**: Nextflow shared-cache options like `$NXF_SINGULARITY_CACHEDIR` and `$NXF_SINGULARITY_LIBRARYDIR` do not apply to ORAS distribution.
- **Offline image availability**: Offline environments require a local Singularity installation to download and use images.
- **Version compatibility**: `oras://` is supported only by recent versions of Singularity and Apptainer.
