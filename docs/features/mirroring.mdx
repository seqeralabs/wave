---
title: Container mirroring
description: Copy a container image to a registry of your choice while preserving its original name and digest.
---

Storing container images in a registry you control improves security, supports policy compliance, and gives faster access. Wave mirroring automates the copy. Wave pulls the image from its source and pushes it to your registry on demand. The image name and digest stay identical to the source.

Wave runs the copy with Skopeo. Tools that pin the image by digest continue to work against the mirrored copy. Mirror operations are asynchronous and dispatched through the job manager.

:::tip
To produce a new, Wave-built image and push it under a name you choose, use [container freeze](./container-freezes.mdx). Freeze returns a new image with a 16-character build-hash tag. Mirroring copies the existing image byte-for-byte with its original name and digest.
:::

## Use cases

Use cases for container mirroring include:

- **Region-specific deployments**: Place containers in the right geographic region to meet data residency requirements or cut latency.
- **Enhanced security**: Serve images from a registry behind a firewall or inside tight networking controls.
- **Cost and performance**: Pull from a cloud-native registry co-located with compute. Egress costs are reduced compared with pulls from remote registries.
- **Multi-cloud pipelines**: Deploy workflows across AWS, Google Cloud, and Azure. Each cloud pulls from its local registry.
- **Mitigating upstream rate limits**: Mirror a public image once and serve many pulls from your own registry.

## How it works

The mirror flow involves the client, Wave, the source registry, and the target registry:

1. A client (Nextflow, the Wave CLI, or the Wave API) requests a container and asks for mirroring to a target registry.
2. Wave validates the request. The source and target registries must differ and the caller must be authenticated.
3. If the target image is already mirrored or in progress, Wave returns the target URI straight away. Otherwise Wave launches a Skopeo copy job that pulls every layer and manifest from the source and pushes them to the target.
4. Wave returns the target registry URI to the client. The client pulls the image directly from the target registry with no further involvement from Wave.
