title: Multi-architecture support
description: Provision containers for linux/amd64 and linux/arm64 on demand.
---

Computational workloads run on different CPU architectures. Wave provisions container images for `linux/amd64` and `linux/arm64` on demand. Pipelines pick the hardware that matches their cost and performance goals.

Use cases include:

- **Cost optimization**: Run pipelines on ARM cloud instances for lower per-hour pricing.
- **Hybrid fleets**: Target the same pipeline at mixed-architecture compute. No separate container catalogues needed.
- **Sustainability**: Reduce energy per task by running on ARM where the workload permits.

## Common architecture types

The two most widely used architectures are `linux/amd64` and `linux/arm64`. Each uses a distinct CPU instruction set.

`linux/amd64`:

- Traditional x86-64 architecture.
- Widely adopted in server, desktop, and HPC environments.

`linux/arm64`:

- 64-bit extension of the ARM architecture.
- Common in modern cloud environments.

## ARM architecture benefits

[ARM processors](https://en.wikipedia.org/wiki/ARM_architecture_family) deliver high performance per watt. That makes them attractive for cloud providers and for teams optimizing cost and sustainability.

## Cloud provider support

Leading cloud providers offer ARM-based compute alongside x86:

- AWS Graviton
- Azure Cobalt
- Google Cloud Axion

ARM adoption is expanding in fields such as bioinformatics and data science. Workloads there are compute-intensive and cost-sensitive.

## Requesting an architecture

Wave clients set the `platform` request parameter to `linux/amd64` or `linux/arm64`. The same Dockerfile or Conda specification builds for either target. The returned image URI resolves to the matching binary.

:::note
`linux/arm64` is not the same as Apple Silicon. Apple Silicon uses a system-on-chip design for macOS. `linux/arm64` is the Linux ARM specification used on server and cloud distributions. Wave does not build images for `macos/arm64`.
:::
